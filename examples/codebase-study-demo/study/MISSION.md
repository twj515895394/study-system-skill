# MISSION：为什么学习 MiniAgentFlow

## 学习主题

MiniAgentFlow 源码学习。

## 背景

用户希望系统读懂一个 Agent 任务编排类项目，而不是只看 README 或零散问几个函数。目标是建立从 CLI 入口、任务解析、调度执行、工具调用到状态持久化的完整心智模型。

## 成功图景

完成本学习工作区后，用户应该能够：

1. 用自己的话解释 MiniAgentFlow 的核心模块和职责边界。
2. 独立追踪一条从 CLI 命令到任务执行的调用链。
3. 判断一个新功能应该改哪个模块，以及可能影响哪些路径。
4. 基于该项目设计一个小扩展，例如新增一个工具或任务类型。

## 暂不学习

1. 不深入性能优化。
2. 不展开部署和生产监控。
3. 不做完整二次开发，只做最小实现练习。

## 诊断结果摘要

用户熟悉 Python 和基础 CLI 工具，但对 Agent 调度系统的状态流和模块边界不够熟悉。第一阶段应先建立项目地图和主调用链，再进入细节扩展。
