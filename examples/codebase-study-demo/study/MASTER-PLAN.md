# MASTER-PLAN：MiniAgentFlow 源码学习主计划

> 这是本学习工作区的唯一权威主计划。其他文件与本文件冲突时，以本文件为准。

## 总目标

系统读懂 MiniAgentFlow 的核心架构、主调用链和可扩展点，并能基于理解做一个最小功能扩展。

## 阶段路线

### Phase 1 - 项目地图与主链路

目标：建立项目目录、模块职责和主调用链的整体认知。

课程：

1. L01：系统总览与核心目录
2. L02：CLI 入口到任务调度的调用链
3. L03：任务状态如何流转

阶段验收：用户能不用看笔记，画出 CLI -> parser -> scheduler -> executor -> state store 的主链路。

### Phase 2 - 核心模块拆解

目标：读懂 scheduler、executor、tool registry、state store 的职责边界。

课程：

4. L04：Scheduler 职责与边界
5. L05：Executor 如何运行任务
6. L06：Tool Registry 如何注册和调用工具
7. L07：State Store 如何保存任务状态

阶段验收：用户能判断一个新功能应该落在哪个模块，并说明风险。

### Phase 3 - 最小扩展练习

目标：把源码理解转成一个可运行的小扩展。

课程：

8. L08：新增一个 echo 工具
9. L09：为任务增加 dry-run 模式
10. L10：阶段复盘与扩展方案设计

阶段验收：用户能说明实现路径、改动文件、影响范围和回滚方案。

## 动态补课规则

如果用户在理解检查中无法解释 Python 包入口、CLI 参数解析、函数调用链或状态对象，则插入补课，但补课必须服务当前课程，并在完成后回到主线。

## 进度原则

不按天数推进。只有理解检查达到 Understanding 及以上，才允许进入下一课；实现类课程建议达到 Skill。
