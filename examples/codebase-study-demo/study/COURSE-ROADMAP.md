# COURSE-ROADMAP：阶段路线图

## Phase 1 - 项目地图与主链路

目标：先知道项目由哪些模块组成，再追一条最重要的调用链。

| 课次 | 主题 | 课型 | 验收方式 |
|---|---|---|---|
| L01 | 系统总览与核心目录 | 架构拆解课 | 画出模块职责表和主链路 |
| L02 | CLI 入口到任务调度的调用链 | 源码调用链课 | 复述 CLI -> scheduler 链路 |
| L03 | 任务状态如何流转 | 源码调用链课 | 解释状态对象在哪里创建、修改、保存 |

## Phase 2 - 核心模块拆解

目标：读懂关键模块的职责边界和扩展点。

| 课次 | 主题 | 课型 | 验收方式 |
|---|---|---|---|
| L04 | Scheduler 职责与边界 | 架构拆解课 | 判断新任务类型应该如何接入 |
| L05 | Executor 如何运行任务 | 源码调用链课 | 解释执行过程和副作用 |
| L06 | Tool Registry 如何注册和调用工具 | 源码调用链课 | 新增一个工具注册点 |
| L07 | State Store 如何保存任务状态 | 架构拆解课 | 说明状态持久化边界 |

## Phase 3 - 最小扩展练习

目标：完成一个小扩展并说明影响范围。

| 课次 | 主题 | 课型 | 验收方式 |

| L09 | 为任务增加 dry-run 模式 | 动手实现课 | 说明改动路径和风险 |
| L10 | 阶段复盘与扩展方案设计 | 阶段复盘课 | 输出扩展方案 |
