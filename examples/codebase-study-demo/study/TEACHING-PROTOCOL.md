# 教学协议与风格规范

## 教学风格定位

- **语言风格**：`[x]` 循序渐进，避免一次塞太多源码细节。
- **讲解重点**：`[x]` 架构与源码结合，先目录和主链路，再深入函数。

## 节奏与概念密度

- 每节课最多引入 3 个新概念。
- 源码课优先使用调用链表格和 Mermaid 流程图。
- 每次讲完一条链路必须做一次理解检查。

## 理解检查偏好

- 优先：调用链复述、改动影响判断、职责边界题。
- 禁止：只问“懂了吗”。

## 用户背景

用户熟悉 Python、命令行工具和基础 Web 开发，可以用 CLI、函数调用、请求处理流程做类比。
