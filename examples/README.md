# Examples

这里放 `study-system` 的完整示例工作区，帮助用户理解这个 Skill 跑起来之后是什么样。

## 示例列表

| 示例 | 场景 | 重点展示 |
|---|---|---|
| `codebase-study-demo/` | 系统学习一个源码项目 | `STATE.json`、主计划、课程进度、源码调用链课、handoff、learning-record、复习计划 |

## 怎么看示例

建议按这个顺序阅读：

1. `examples/codebase-study-demo/README.md` —— 示例说明。
2. `examples/codebase-study-demo/study/STATE.json` —— 当前机器可读状态。
3. `examples/codebase-study-demo/study/CURRENT.md` —— 人类可读当前状态。
4. `examples/codebase-study-demo/study/MASTER-PLAN.md` —— 学习主线。
5. `examples/codebase-study-demo/study/lessons/L01-system-overview.md` —— 一节已经完成的示例课。
6. `examples/codebase-study-demo/study/handoffs/2026-06-25-L01-system-overview.md` —— 跨会话交接单。

## 说明

示例中的项目名、文件路径和代码结构都是虚构的，用来展示工作区组织方式和文档写法。不要把它当成真实仓库的源码分析结论。
