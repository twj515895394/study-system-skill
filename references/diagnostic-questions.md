# 诊断问题库

按领域挑 3-5 个最相关的问题，不必问满。目的是判断：用户为什么学、已有基础、是否具备前置知识、学习目标偏应用还是理论、期望节奏、资料完整度、最终产出。

挑选原则：优先挑"答案会改变你接下来怎么搭工作区"的问题。比如用户已经在第一句话里说清楚了目标和产出，就不用再问对应的问题，直接确认即可——诊断是为了补齐缺失信息，不是走过场。

## 通用问题模板（适用所有领域，缺省兜底）

1. 你为什么要学这个？最终希望解决什么现实问题？
2. 你目前对这个主题了解多少？能否举一个你已经知道的核心概念？
3. 这门课/这个项目需要哪些前置基础？你觉得自己哪些具备，哪些薄弱？
4. 你的目标是"理解原理""能上手操作""能做项目""能考试""能教别人"，还是其他？
5. 你希望学习节奏偏快还是偏稳？遇到难点时希望深入拆解还是先跳过？
6. 你手头有哪些完整资料？教材、课程、文档、源码、论文、视频、题库、SOP是否齐全？
7. 学习结束时，你希望产出什么？笔记、项目、方案、考试成绩、作品集、能力证明？
8. 有没有明确不想学、暂时不需要学、或容易分散精力的相邻内容？

## 编程项目（domain=code）

1. 你学这个项目是为了使用、二开、面试、架构借鉴，还是企业落地？
2. 你熟悉该项目使用的语言和框架吗？
3. 你更想先看整体架构，还是直接看核心调用链？
4. 你是否需要补基础，例如协议、框架、数据库、消息队列、前端、部署？
5. 最终希望产出源码阅读报告、架构图、二开方案，还是可运行 Demo？

## 专业课/教材（domain=course）

1. 这门课是为了考试、竞赛、升学，还是实际应用？
2. 目前学到哪一章？最薄弱的是概念、公式、题型还是应用？
3. 有没有教材、讲义、课后题、历年题？
4. 希望先补基础，还是跟着学校进度同步提高？
5. 每次学习后是否需要练习题和错题记录？

## 行业知识（domain=industry）

1. 你学习这个行业是为了入门、转岗、管理决策、产品设计，还是投资研究？
2. 你目前是否了解行业基本术语和上下游链路？
3. 手头资料是报告、SOP、培训资料、政策文件、案例，还是内部文档？
4. 更希望学业务流程、商业模式、岗位技能，还是行业趋势？
5. 最终希望产出行业地图、业务流程图、竞品分析，还是行动方案？

## 考试备考（domain=exam）

1. 考试时间是什么时候？还有多久？
2. 目前的薄弱环节是知识点、题型、速度，还是心态？
3. 有没有历年题、错题本、模拟卷？
4. 希望按章节系统过一遍，还是按题型专项突破？
5. 每天/每周能投入多少时间？

## 孩子学科辅导（domain=child）

1. 孩子目前在学校的进度到哪了？最近的考试/作业反映出哪些薄弱点？
2. 是想跟学校进度同步巩固，还是要提前学/拉开差距？
3. 孩子的学习习惯是什么样的（专注时长、喜欢的讲法、容易卡在哪类题）？
4. 家长希望多大程度参与（旁听、检查、还是完全交给孩子自己跟）？
5. 有没有教材、练习册、老师布置的资料？
